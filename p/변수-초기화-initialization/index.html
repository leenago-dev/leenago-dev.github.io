#The error occurs because region (and salary) are only assigned inside the if len(categories) >= 2: block, but they're used outside of it. If there are fewer than 2 categories, these variables won't be defined.

#Here's the fix - initialize the variables before the if statement:

# ---
# // ... existing code ...

    categories = job.find_all("p", class_="new-listing__categories__category")  
    # There are more than two categories, so we need to use 'find_all'

    salary = "N/A"  # Default value
    region = "N/A"  # Default value

    if len(categories) >= 2:
        salary_html, region_html = categories[-2:]
        salary = salary_html.text.strip()
        region = region_html.text.strip()

# // ... existing code ...

그리고 드는 의문 하나, 왜 ‘default value’ 값을 먼저 넣어줘야 하는거지?

그 이유는…

어떤 참석자의 ‘salary’와 ‘region’ 이름표가 누락됐을 때를 대비해서, 일단 모든 참석자의 이름표를 ‘정보 없음’으로 만들어두기위함입니다.

아니 이게 무슨소리냐 싶으실텐데요ㅋㅋㅋ 차근차근 알아가봅시다!

  1. 참석자 명단을 한 명씩 확인합니다.

		salary = salary_html.text.strip()

그런데 만약, 위 과정에서 어떤 참석자가 2번을 빠트렸다면 어떻게 될까요?

  • 정보가 누락된 참석자의 이름표가 존재하지 않기 때문에 NameError가 발생하면서 코드가 더이상 실행되지 않습니다.

▶️ 그렇기 때문에, salary = None으로 일단 모든 참석자의 이름표를 ‘정보 없음’으로 생성해두는 것입니다.

  • 만약 신청자의 정보가 있다면, 3번 과정에서 실제 정보로 내용을 덮어쓰게 되는거죠!
    • 이것이 바로 방어적 프로그래밍 입니다.

결론

  • Initialization은 데이터가 불규칙할 때 프로그램이 중간에 멈추는 것을 막아주는 안전장치이다!

오늘의 용어사전

  • 초기화 (Initialization): 변수를 처음 만들 때, 초기의 기본값을 할당해주는 과정
    • ex) salary = None
  • 방어적 프로그래밍 (Defensive Programming): 데이터가 없거나 형식이 다를 예외적인 경우를 미리 예상하고, 에러가 나지 않도록 코드를 안전하게 작성하는 프로그래밍 스타일
